Understanding Permissions

Permissions define what each user has access to within the system. Each permission has a different level of access, with each level being defined as a role on the Roles screen in System Config mode.

For each role, an administrator will decide what privileges the users will have access to. For example, one role may have access to one mode in the mode switcher, while another role has access to all modes. Thess roles then appear on the Permissions screen, where you can assign which users or user groups have access to the set of privilegs.
